PART 2 — BACKEND: UPDATE THE N8N PAYLOAD
--------------------------------------------------------------------

FILE: backend/internal/services/ai_service.go

FUNCTION: callN8NRoadmap

CURRENT BEHAVIOR: sends one mode ("weekly" | "topic" | "interview")
and weekly_hours is hardcoded to 15.

NEW BEHAVIOR:
  - weekly_hours must come from sc.Goals.WeeklyHours (default 15 if nil)
  - The mode field in the payload should be "all" — this tells the
    n8n agent to return all three modes (topic_mode, weekly_mode,
    interview_mode) in one unified JSON response
  - Keep all existing CF + LC fields exactly as they are

The EXACT payload shape the n8n agent expects is:

    },
    "leetcode": {
      "total_solved": 247,
      "easy":          98,
      "medium":       132,
      "hard":          17,
      "topics": {
        "Array":              45,
        "String":             38,
        "Hash Table":         42,

NOTE: The "topics" map for Codeforces comes from sc.CFTagFreq
(map[string]int, built from verdict=="OK" submissions → tag counts).

(map[string]int, built from the /skill endpoint tiers).
Both are already populated in BuildStudentContext — do not change
that function. Only change the payload assembly in callN8NRoadmap.

ALSO UPDATE: The GenerateRoadmap function currently passes mode as a
string from the HTTP request. Change it so mode is always sent as
"all" to the n8n agent. The frontend mode selector (weekly/topic/
interview) now controls which section of the RESPONSE to display,
not which mode is requested from the agent.

FILE: frontend/src/types/index.ts

Add these TypeScript interfaces (or update existing ones):

  interface RoadmapSummary {
    total_weeks:      number
    estimated_hours:  number
    focus_areas:      string[]
    milestones:       Array<{ week: number; description: string }>
  }

  interface RoadmapProblem {
    title:      string
    platform:   string
    url:        string
    rating:     number | null
    difficulty: string | null
    tags:       string[]
    reason:     string
  }

  interface TopicSection {
    name:           string
    why:            string
    strength_score: number
    problems:       RoadmapProblem[]
  }

  interface WeekSection {
    week:         number
    theme:        string
    focus_topics: string[]
    problems:     RoadmapProblem[]
  }

  interface InterviewPattern {
    name:            string
    frequency:       number
    user_strength:   string
    problems_solved: number
    problems:        RoadmapProblem[]
  }

  interface UnifiedRoadmap {
    summary:        RoadmapSummary
    topic_mode:     { generated_at: string; goal_summary: string; topics:   TopicSection[]     }
    weekly_mode:    { generated_at: string; goal_summary: string; weeks:    WeekSection[]       }
    interview_mode: { generated_at: string; target_companies: string[]; patterns: InterviewPattern[] }
  }

SUMMARY SECTION (render above mode tabs):

  A panel showing:
  - "Total weeks: {summary.total_weeks}"
  - "Estimated hours: {summary.estimated_hours}"
  - Focus areas as pills: {summary.focus_areas.map(area => <span>)}
  - Milestones as a small list:
      Week {m.week}: {m.description}

  Style: use var(--panel), var(--accent), var(--text-dim).
  Keep it compact — single row on wide screens, stacked on narrow.

TOPIC MODE (roadmap.topic_mode.topics):

  For each topic:
  - Topic name as section header
  - "why" text in var(--text-dim)
  - Strength bar: width = strength_score * 100%,
    colour = lerp from var(--err) at 0.0 to var(--ok) at 1.0
    Use inline style: oklch values — low=red, mid=warn, high=ok
  - Problem list (same ProblemRow component already in the file)

WEEKLY MODE (roadmap.weekly_mode.weeks):

  Same as current weekly rendering. Each week is a collapsible
  accordion showing: week number, theme, focus_topics as pills,
  progress bar placeholder, then problem rows.

INTERVIEW MODE (roadmap.interview_mode.patterns):

  For each pattern:
  - Pattern name + frequency bar (frequency * 100% width)
  - user_strength badge: "weak" = var(--err), "moderate" = var(--warn),
    "good" = var(--ok)
  - problems_solved count
  - Problem list (same ProblemRow)

GENERATE BUTTON BEHAVIOR:

  When the user clicks "Generate Roadmap", POST /roadmap/generate
  with body { "mode": "all" }. On success, parse and store the
  full unified JSON. All three mode tabs become available immediately.

  If the stored roadmap was generated with the old format (no
  "summary" key), show a "Regenerate" banner prompting the user
  to regenerate.
