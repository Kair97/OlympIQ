## AUTHENTICATION

OlympIQ uses httpOnly cookie-based JWT auth.
n8n cannot use cookies directly — for internal automation, add a dedicated
API key middleware OR use the service account login flow:

Step 1 — Login to get session cookies:
  POST /auth/login
  Body: { "email": "admin@olympiq.com", "password": "your-password" }
  Response: sets access_token + refresh_token as httpOnly cookies

Step 2 — In all subsequent n8n HTTP Request nodes:
  - Enable "Send Cookies" option
  - OR extract the access_token from the login response Set-Cookie header
    and send it as: Cookie: access_token=<value>

Recommended: Create a dedicated internal admin account for n8n automation.

## ENDPOINTS N8N SHOULD CALL

### 1. Health check (no auth needed)
  GET /health
  Use in: Docker Error Watcher — check if backend is up

### 2. Trigger stats sync for all users
  POST /accounts/sync
  Auth: required
  Use in: After Session Brain Update — refresh user data after code changes

### 3. Get dashboard data (verify data looks correct)
  GET /dashboard
  Auth: required
  Use in: Morning Briefing — read user stats to include in briefing

### 4. Get roadmap
  GET /roadmap
  Auth: required
  Use in: Morning Briefing — show what's in the current roadmap

### 5. Get recommendations
  GET /recommendations
  Auth: required
  Use in: Morning Briefing — show today's suggested problems

### 6. Test AI connection
  GET /ai/test
  Auth: required
  Use in: Docker Error Watcher — check if Gemini API is responding
